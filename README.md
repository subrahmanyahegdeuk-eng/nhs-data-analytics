NHS Lung Cancer Screening — Report Generator

An AI-powered data analytics tool that transforms raw NHS 
lung cancer screening data into a complete monthly 
performance report in seconds including interactive 
dashboards, regional breakdowns, data quality checks, 
and an AI-generated executive summary written in plain 
English for non-technical Programme Directors.

The Problem This Solves

NHS screening programme analysts spend hours every month 
manually producing performance reports — cleaning raw data, 
running queries, building charts, and writing narrative 
summaries for programme managers. This tool automates the 
entire process. Upload a CSV and get a complete report in 
under 30 seconds.

What It Does
Upload any NHS screening dataset and instantly receive:

Performance Metrics**
Total patients referred, attended, and did not attend
Overall completion rate and DNA (Did Not Attend) rate
Total positive screening results

Visual Dashboard
Monthly completion rate trend chart with average baseline
Regional performance bar chart highlighting above and 
below average regions in green and red

Completion rate by age group bar chart

Detailed Analysis Tables
Monthly summary with referrals, attendance, and completion rates
Regional breakdown with positive result counts
Screening outcomes with percentage breakdown
DNA reasons analysis
Site-level performance ranked by completion rate
Automated data quality report flagging missing values, duplicate IDs, out-of-range ages, and invalid dates

AI Executive Summary**
4-paragraph plain English narrative generated by LLM
Covers overall performance, regional highlights, key risks, and recommended actions
Written for non-technical Programme Directors — no data background required to understand it

Export Options
Full 8-tab formatted Excel report download
Executive summary text file download

Why This Matters for NHS

Data accuracy in cancer screening directly affects patient 
outcomes. Late detection of lung cancer significantly reduces 
survival rates. This tool helps screening programmes:

Identify underperforming regions before month-end reviews
Flag data quality issues before they affect reporting
Give programme managers instant insight without waiting for analyst availability
Reduce time spent on routine reporting so analysts can focus on higher-value work

SQL Skills Demonstrated

This project uses 7 analytical SQL queries covering:

sql
Monthly performance trend
SELECT strftime('%Y-%m', referral_date) as month,
    COUNT(*) as total_referred,
    ROUND(SUM(CASE WHEN attended='Yes' 
        THEN 1.0 ELSE 0.0 END)/COUNT(*)*100,1) 
        as completion_rate
FROM screening
GROUP BY month ORDER BY month

Regional completion rates
SELECT region,
    ROUND(SUM(CASE WHEN attended='Yes' 
        THEN 1.0 ELSE 0.0 END)/COUNT(*)*100,1) 
        as completion_rate
FROM screening
GROUP BY region ORDER BY completion_rate DESC

Data quality validation
SELECT COUNT(*) as total_records,
    SUM(CASE WHEN patient_id IS NULL 
        THEN 1 ELSE 0 END) as missing_id,
    SUM(CASE WHEN age < 55 OR age > 74 
        THEN 1 ELSE 0 END) as out_of_range_age
FROM screening

Data

This project uses a realistic synthetic dataset of 2,000 
patient records generated to mirror real NHS lung cancer 
screening data structures. The dataset includes:

Patient demographics (age 55-74, gender)
Referral and screening dates
9 NHS regions and 27 screening sites
Attendance status and DNA reasons
Screening outcomes (Negative, Positive Low/High Risk, Inconclusive)
Distance from screening site
Previous DNA history

No real patient data is used. The synthetic data was 
designed to reflect realistic attendance patterns patients with previous DNA history, greater distances, 
and older age groups show statistically higher non-attendance rates.
